NEXT STEPS TO RUN THE SYSTEM:
==============================================

1. START DOCKER DESKTOP
   - Open Docker Desktop application
   - Wait for it to fully start
   - Verify: docker ps should work

2. BUILD AND START THE SYSTEM
   
   PowerShell:
   .\scripts\quick_start.ps1
   
   Or manually:
   docker-compose -f docker-compose-full.yml build
   docker-compose -f docker-compose-full.yml up -d

3. START TEST CLIENT
   
   docker-compose -f docker-compose-full.yml --profile test up -d test-client

4. RUN TESTS
   
   .\scripts\run_tests.ps1
   
   Or manually:
   docker exec test-client /app/bin/distributed_client frontend-1:60051 mkdir / videos
   docker exec test-client /app/bin/distributed_client frontend-1:60051 create /videos test.mp4
   docker exec test-client /app/bin/distributed_client frontend-1:60051 write /videos/test.mp4 "Hello!"
   docker exec test-client /app/bin/distributed_client frontend-1:60051 read /videos/test.mp4

==============================================
TROUBLESHOOTING:
==============================================

If Docker is not running:
  - Start Docker Desktop
  - Wait for the whale icon to be stable
  - Run: docker ps (should work without error)

If build fails:
  - Clean: docker-compose -f docker-compose-full.yml down -v
  - Rebuild: docker-compose -f docker-compose-full.yml build --no-cache

If tests fail:
  - Check logs: docker logs frontend-1
  - Check storage: docker logs storage-node-1
  - Check metadata: docker logs metadata-node-1
